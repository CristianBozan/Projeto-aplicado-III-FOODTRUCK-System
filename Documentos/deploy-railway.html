🚀 Guia de Deploy — Food Truck System

Projeto Aplicado III · Equipe 10 · SENAI SC

v3.1 · Railway + MySQL
Este documento descreve o passo a passo para colocar o Food Truck System v3.1 em produção utilizando o Railway como plataforma de hospedagem gratuita, com backend Node.js e banco de dados MySQL na nuvem.
1

Criar conta no Railway

Acesse railway.app e crie uma conta gratuita usando o GitHub.

  1. Acesse https://railway.app
  2. Clique em "Login" → "Login with GitHub"
  3. Autorize o acesso do Railway à sua conta GitHub
ℹ️ O plano gratuito do Railway oferece $5 de crédito/mês, suficiente para este projeto rodar continuamente.
2

Criar o projeto e conectar o repositório

  1. No painel do Railway, clique em "New Project"
  2. Selecione "Deploy from GitHub repo"
  3. Escolha o repositório: CristianBozan/Projeto-aplicado-III-FOODTRUCK-System
  4. Clique em "Deploy Now"

O Railway detecta automaticamente o package.json e executa npm start.

⚠️ O deploy vai falhar neste momento pois o banco ainda não está configurado. Isso é esperado — continue para o passo 3.
3

Adicionar banco de dados MySQL

  1. Dentro do projeto no Railway, clique em "+ New"
  2. Selecione "Database" → "MySQL"
  3. Aguarde o banco ser provisionado (cerca de 30 segundos)
  4. Clique no serviço MySQL → aba "Connect"
  5. Anote os dados de conexão: Host, Port, User, Password, Database
✅ O Railway injeta a variável DATABASE_URL automaticamente no serviço Node.js quando o banco é criado no mesmo projeto.
4

Criar as tabelas no banco de dados

Utilize o arquivo sql/schema.sql do repositório para criar todas as tabelas.

Opção A — Via terminal (recomendado):

mysql -h SEU_HOST -P SEU_PORT -u SEU_USER -p SEU_DATABASE < sql/schema.sql

Opção B — Via Query Editor do Railway:

  1. No painel do MySQL, clique na aba "Data"
  2. Clique em "Query"
  3. Cole o conteúdo do arquivo sql/schema.sql e execute

Após executar, verifique se as tabelas foram criadas:

SHOW TABLES;

Resultado esperado:

atendentes
backups
estoque_logs
itens_pedido
mesas
pedidos
produtos
sincronizacoes
vendas
5

Configurar variáveis de ambiente

No painel do serviço Node.js no Railway, clique em "Variables" e adicione:

Variável Valor Descrição
NODE_ENV production Ativa modo produção (SSL no banco)
JWT_SECRET chave longa e aleatória Segredo para assinar tokens JWT
JWT_EXPIRES 8h Validade do token de sessão
ADMIN_LOGIN superadmin Login do Super Admin
ADMIN_SENHA sua senha Senha do Super Admin
GERENTE_LOGIN admin Login do Gerente
GERENTE_SENHA sua senha Senha do Gerente
ATENDENTE_LOGIN funcionario Login do Atendente
ATENDENTE_SENHA sua senha Senha do Atendente
BACKUP_TOKEN token secreto Token de proteção dos backups
⚠️ Não adicione DATABASE_URL manualmente. O Railway já injeta essa variável automaticamente quando o banco MySQL está no mesmo projeto.
6

Redeploy e acesso ao sistema

  1. Após configurar as variáveis, clique em "Deploy" (ou o Railway faz automaticamente)
  2. Aguarde o build finalizar (~2 minutos)
  3. Na aba "Settings" do serviço, clique em "Generate Domain" para obter a URL pública
  4. Acesse a URL gerada no navegador

Credenciais padrão para o primeiro acesso:

PapelUsuárioSenha padrão
Super Adminsuperadmindefinida no ADMIN_SENHA

✅ O sistema estará online e acessível pelo celular ou computador de qualquer lugar com internet.
7

Verificar se está funcionando

Acesse o endpoint de health check para confirmar que o servidor está rodando:

GET https://SUA-URL.railway.app/auth/ping

Resposta esperada:

{ "status": "ok", "ts": "2026-04-13T..." }

Teste o login via terminal (opcional):

curl -X POST https://SUA-URL.railway.app/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"sua_senha"}'
